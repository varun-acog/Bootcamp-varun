Images Exercise Notes

Level 1: Basics of Images

Approach:

  • Added three images to a webpage using the `` tag.
  • Styled the images with CSS properties such as `width`, `height`, and `border-radius` to make the images visually appealing.
  • Used a placeholder image service (Lorem Picsum) to generate one of the images.

Key Learnings:

  • Understanding the differences between image formats: JPEG (lossy compression), PNG (lossless, transparency), and SVG (scalable vector graphics).
  • CSS allows for basic customization of image properties to control their size, shape, and layout.
  • Placeholder image services are helpful for rapid prototyping.

Challenges Faced:

  • Ensuring images maintained aspect ratios when resized.
  • Choosing appropriate dimensions to balance quality and loading performance.

Level 2: Advanced Image Styling

Approach:

  • Edited an image using a tool like Photoshop to reduce its size and compress it for faster loading.
  • Added a full-bleed image to the top of the page as a header using CSS properties like `width: 100%` and `object-fit: cover`.
  • Used CSS overlays, including gradient and color tint, on images to enhance their visual appeal and improve text readability.

Key Learnings:

  • Compression and resizing significantly reduce image load times without noticeable quality loss.
  • Full-bleed images create an immersive effect, especially when paired with overlay techniques.
  • CSS gradients can enhance images for aesthetic purposes and improve readability of overlaid text.

Challenges Faced:

  • Maintaining image quality while compressing files for performance optimization.
  • Ensuring the full-bleed image looked consistent across devices and resolutions.

Level 3: Responsive Images and Optimization

Approach:

  • Implemented responsive images using the `` element to serve different versions of an image based on screen size.
  • Created high-resolution and low-resolution versions of an image for different devices.
  • Used lazy loading for images to improve initial page load speed by only loading images when they are about to appear in the viewport.

Key Learnings:

  • The `` element and `srcset` attribute are powerful tools for delivering device-specific images.
  • Lazy loading improves performance, especially on pages with multiple images, by reducing the initial load time.
  • Optimizing images for different devices is essential for maintaining a balance between quality and performance.

Challenges Faced:

  • Creating multiple image versions and ensuring correct fallback behavior for older browsers.
  • Testing lazy loading behavior across different browsers and devices.
  • Balancing performance optimization with the need for high-quality visuals on retina displays.
